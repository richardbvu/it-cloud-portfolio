## Projects

### [Active Directory User Lifecycle](https://github.com/richardbvu/active-directory-user-lifecycle)
On-prem Active Directory lab focused on user lifecycle management, group-based access control, and workstation domain join workflows, with validation of authentication, policy application, and troubleshooting scenarios.

**Key Focus Areas:**
- Created and managed domain user accounts, including provisioning, group assignment, and lifecycle changes (enable, disable, password reset)  
- Assigned access using security groups and validated permission behavior across user sessions  
- Joined a Windows workstation (W11VM) to the domain and verified domain connectivity and authentication  
- Validated Group Policy application and distinguished between user and computer policy behavior  
- Performed account and workstation troubleshooting, including login issues, domain trust checks, and policy verification  

---

### [Microsoft Entra Identity And Access Operations](https://github.com/richardbvu/microsoft-entra-identity-and-access-operations)
Cloud-based Microsoft Entra lab focused on user lifecycle management, access control, and authentication behavior, including RBAC, Administrative Units, and sign-in troubleshooting.

**Key Focus Areas:**
- Created and managed cloud-only users, including account provisioning, profile updates, password resets, and sign-in state control  
- Configured security groups and validated membership behavior across assigned and dynamic group types  
- Applied Role-Based Access Control (RBAC) concepts to understand permission boundaries and administrative roles  
- Configured Administrative Units and evaluated how scoped administration affects user and group management  
- Investigated user sign-in issues by validating account status, User Principal Name (UPN) format, and authentication behavior  
- Verified identity and access behavior through portal-based checks and controlled testing of user actions  

---

### [Hybrid Identity Foundations And Source Of Authority](https://github.com/richardbvu/hybrid-identity-foundations-and-source-of-authority)
Hybrid identity lab integrating Active Directory and Microsoft Entra ID, focused on synchronization behavior, source of authority, identity consistency, and cross-environment troubleshooting.

**Key Focus Areas:**
- Validated hybrid identity synchronization between on-prem Active Directory and Microsoft Entra ID, confirming how user objects are represented across both environments  
- Identified and analyzed source of authority for user accounts, distinguishing between on-prem managed and cloud-managed attributes  
- Compared User Principal Name (UPN) values between Active Directory and Entra to evaluate sign-in consistency and identity alignment  
- Investigated identity-related issues caused by mismatched attributes, including incorrect UPN configuration and changes made in the wrong environment  
- Evaluated device identity and join state to understand how domain-joined systems interact with cloud identity services  
- Performed cross-environment troubleshooting to determine whether issues originated in Active Directory, Entra ID, or the synchronization layer
